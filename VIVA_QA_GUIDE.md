## Q1: "If the Blockchain is Public, how is the Bid Encrypted? Can't everyone see it?" ðŸ”’

**The Analogy: The Glass Safe**
*   Imagine a transparent glass safe (Blockchain). Everyone can see *that* there is an envelope inside, but they cannot read the letter inside the envelope because it is inside a **Locked Metal Box** (The Hash/Encryption).

**The Technical Answer (Commit-Reveal)**:
*"We use a **Commit-Reveal Scheme**. The Vendor submits a cryptographic hash of their price. The Blockchain stores `0x88a...` (Gibberish). The Officer's function is to call `openTender()` which allows the Salt to be accepted, mathematically proving the original number."*

## Q10: "You said Blockchain is a Tree/Chain of relations. Where is that in your code?" ðŸŒ³â›“ï¸

**The Doubt**: Where is the "Linked List" or "Tree" data structure?

**The Answer (Protocol vs Application)**:
1.  **The Chain (Block Linking)**:
    *   **Concept**: Block 50 contains the unique "Fingerprint" (Hash) of Block 49.
    *   **Relation**: `Block_50.PreviousHash == Hash(Block_49)`.
    *   **Implementation**: **Hardhat** does this automatically. If a hacker tries to change a Bid in Block 49, its hash changes, and Block 50 will *reject* it because the link is broken.

2.  **The Tree (Merkle Tree)**:
    *   **Concept**: How do we fit 1000 Transactions (Bids) into 1 Block? We use a **Merkle Tree**.
    *   **Relation**: `Hash(Bid A) + Hash(Bid B) -> Hash(AB)`. This continues up to a single "Root Hash".
    *   **Your Project**: When `AgriProcure.sol` emits an event, it becomes a "Leaf" in this tree.
    *   **Visual Proof**: In the **Auditor Dashboard**, when you see "Integrity: 100% Match", it means the Auditor has re-calculated this Tree path and found it valid.

**Summary**: Hardhat builds the Tree/Chain. Our `AuditorDashboard` verifies it.

## Q15: "Why not fully automate payment based on feedback? (The Pure Crypto Model)" ðŸ¤–ðŸ’”

**The Answer (The Digital vs. Physical Gap)**:

*   **Verdict**: We cannot use "Pure Crypto Logic" for "Non-Digital Assets". We *must* have a **Trusted Oracle** (The Officer + IoT Sensors) to confirm reality. Code handles the *Money*, Humans handle the *Matter*.
